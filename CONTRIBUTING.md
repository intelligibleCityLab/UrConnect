# Contributing to UrConnect

Thank you for helping improve UrConnect. This project is preparing for public open-source development, so the contribution process is intentionally simple.

[简体中文](CONTRIBUTING.zh-CN.md)

## Reporting Issues

Use GitHub issues for reproducible bugs, feature requests, build problems, and documentation gaps. Include:

- UrConnect version or commit hash
- Operating system, compiler, CMake, Qt, and Boost versions
- Steps to reproduce
- Expected and actual behavior
- Minimal input data when redistribution is allowed

Please do not upload private or restricted GIS datasets.

## Pull Requests

Before opening a pull request:

- Keep changes focused on one bug, feature, or documentation task.
- Follow the existing C++ and CMake style in the surrounding files.
- Update English documentation first; add Chinese documentation when the user-facing text changes.
- Avoid committing generated binaries, local build outputs, IDE files, or machine-specific paths.
- Explain how the change was tested.

## Licensing

By contributing, you agree that your contribution will be licensed under the repository's GPLv3-compatible terms. Do not copy code, datasets, icons, or documents into the repository unless their license allows redistribution under compatible terms.
