# 为 UrConnect 做贡献

感谢你帮助改进 UrConnect。本项目正在准备公开开源，因此贡献流程先保持简洁。

默认英文说明见 [CONTRIBUTING.md](CONTRIBUTING.md)。

## 提交问题

请使用 GitHub issues 反馈可复现 bug、功能请求、构建问题和文档缺口。请尽量提供：

- UrConnect 版本或 commit hash
- 操作系统、编译器、CMake、Qt、Boost 版本
- 复现步骤
- 预期行为和实际行为
- 在允许再分发时提供最小输入数据

请不要上传私有或受限制的 GIS 数据集。

## Pull Request

提交 pull request 前请确认：

- 每个变更聚焦于一个 bug、功能或文档任务。
- 遵循周边文件已有的 C++ 和 CMake 风格。
- 优先更新英文文档；用户可见文本变化时同步补充中文文档。
- 不提交生成二进制、本地构建产物、IDE 文件或个人机器路径。
- 说明变更经过了怎样的测试。

## 许可证

提交贡献即表示你同意你的贡献以本仓库 GPLv3 兼容条款发布。不要复制许可不兼容或再分发权限不明确的代码、数据集、图标或文档进入本仓库。
